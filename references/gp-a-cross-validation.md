# GP-A 周期涨跌幅交叉验证规范

> v2.7.1 新增。本文档收录 GP-A 数据异常的实盘案例、交叉验证方法论、以及已知局限性。

---

## 问题背景

腾讯行情API (`qt.gtimg.cn`) 的 `GP-A` 区段提供四段预计算周期涨跌幅：

| 字段索引 | 含义 | fetch_quote.py 键名 |
|----------|------|---------------------|
| fields[62] | 近1月涨跌幅 | `gp_a_month` |
| fields[63] | 近3月涨跌幅 | `gp_a_3month` |
| fields[64] | 近6月涨跌幅 | `gp_a_6month` |
| fields[65] | 近1年涨跌幅 | `gp_a_year` |

**问题**：这些值是腾讯服务端预计算的，并非实时从K线推算。当股价经历快速大幅波动时，GP-A值的更新可能滞后于实际价格变化，导致严重偏差。

---

## 实盘案例：巨化股份 600160（2026-07-03）

| 周期 | API GP-A | K线实算 | 偏差 | 判定 |
|------|----------|---------|------|------|
| 近1月 | +30.35% | +28.48% | 1.87pp | ✅ 通过 |
| 近3月 | **-3.14%** | **+48.20%** | **51.34pp** | ❌ 替换 |
| 近6月 | **+1.13%** | **+32.77%** | **31.64pp** | ❌ 替换 |
| 近1年 | **+19.21%** | **+77.65%** | **58.44pp** | ❌ 替换 |

**4个周期中3个严重失准，最大偏差 58pp。** -3.14% vs +48.2% 是定性错误——把大涨报成微跌。

---

## 数据流转（定位根因用）

```
腾讯API qt.gtimg.cn
  → fields[62]-[65] = GP-A预计算值（不可信）
    → fetch_quote.py 原样提取
      → H5/报告展示 ← 如果没有交叉验证，错误数据直接到这里

正确流程：
腾讯API qt.gtimg.cn
  → fields[62]-[65] = GP-A原始值
    → fetch_quote.py --cross-check
      → ifzq K线独立计算 (qfq前复权)
        → 逐段对比，偏差 > 5pp → 自动替换为K线值
          → 输出 gp_cross_check.status + corrected 值
            → H5/报告使用替换后的值 ✅
```

---

## 交叉验证方法

`fetch_quote.py --cross-check` 的内部流程：

1. **拉GP-A原始值**：`fields[62]-[65]`
2. **拉K线数据**：`ifzq API`，300日，qfq前复权
3. **独立计算周期涨跌**：
   - 从K线最新日期回推：30天 / 90天 / 180天 / 365天
   - 找最接近目标日期的K线记录
   - 计算：`(latest_close - base_close) / base_close × 100`
4. **逐段对比**：|API值 - K线值| > 5pp → 替换
5. **输出**：
```json
{
  "gp_cross_check": {
    "status": "pass" | "fail" | "unverified",
    "checks": {
      "gp_a_3month": {
        "label": "近3月",
        "api": -3.14,
        "kline": 48.20,
        "diff": 51.34,
        "pass": false,
        "base_date": "2026-04-07",
        "base_close": 33.30,
        "latest_date": "2026-07-03",
        "latest_close": 49.35
      }
    },
    "corrected": {
      "gp_a_month": 30.35,
      "gp_a_3month": 48.20,
      "gp_a_6month": 32.77,
      "gp_a_year": 77.65
    },
    "summary": "⚠️ GP-A交叉验证失败！3/4个周期偏差超5.0pp..."
  }
}
```

**阈值设计**：5个百分点。非对称——宁可漏掉小偏差（<5pp），也不误杀正常数据。GP-A在正常行情下精度通常在2pp以内。

## 已知局限性

### 分红股假阳性

K线数据使用 **qfq（前复权）**，历史价格会因分红向下调整。GP-A可能使用不复权价格。

- **巨化 600160**（近无分红）：qfq最新收=实时行情，无调整偏差 → ✅ 验证100%可靠
- **茅台 600519**（有大额分红）：qfq调整使历史价格偏低 → 计算的涨跌比GP-A更负面 → 可能触发假阳性

**当前处理**：假阳性不影响数据准确性——K线计算值替代GP-A后，报告使用前复权口径的涨跌。假阳性的代价仅是标记了"API异常"，数据本身是正确的。

**未来改进方向**：检测分红事件，对分红股使用更宽阈值（8-10pp）或尝试获取不复权K线。
